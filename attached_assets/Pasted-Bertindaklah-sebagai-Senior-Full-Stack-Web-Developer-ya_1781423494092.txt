Bertindaklah sebagai Senior Full-Stack Web Developer yang ahli dalam React.js, Tailwind CSS, dan integrasi REST API. Buatkan Aplikasi Web "Reed Stock Management" (Manajemen Stok Sisir Mesin Tenun) yang responsif dengan pendekatan mobile-first design dan tema Dark Mode (UI modern dan rapi seperti Glide Apps).
вЂ‹Tech Stack yang Dibutuhkan:
вЂ‹Framework: React.js (dengan react-router-dom untuk navigasi).
вЂ‹Styling: Tailwind CSS (Tema Dark Mode: background gelap, teks terang, aksen warna solid hijau/merah/biru).
вЂ‹Icons: lucide-react.
вЂ‹PDF Generator: jspdf dan jspdf-autotable.
вЂ‹State Management & API: React Hooks & Fetch/Axios.
вЂ‹Database & API Integrations:
Database menggunakan Google Sheets via SheetDB API.
URL Endpoint Utama: [https://sheetdb.io/api/v1/jsrzvylcy75sj](https://sheetdb.io/api/v1/jsrzvylcy75sj)
Terdapat 5 sheet terpisah. Buatkan service wrapper terpusat untuk CRUD ke endpoint berikut (berdasarkan parameter ?sheet=nama_sheet):
вЂ‹DASHBOARD: Metrik summary (Total Mesin, Stok, dll).
вЂ‹MASTER_STOK: Master data sisir (ID, Spesifikasi/Dimensi, Status).
вЂ‹LIVE_TRACKING: Data mesin, lokasi, dan ID sisir yang sedang terpasang.
вЂ‹HISTORY_PASANG: Log khusus pemasangan sisir.
вЂ‹HISTORY_LEPAS: Log khusus pelepasan dan service sisir (termasuk kondisi akhir).
вЂ‹Kebutuhan 4 Tab Utama (Gunakan Bottom Navigation Bar):
вЂ‹1. Tab: Dashboard
вЂ‹Summary Cards: Ambil data dari DASHBOARD untuk menampilkan kartu metrik (Total Mesin Aktif, Kosong, Stok di Gudang, Sisir Rusak).
вЂ‹Live Status: Ambil data dari LIVE_TRACKING. Tampilkan Grid/List ringkas kondisi semua mesin saat ini. Beri indikator (Misal: Border/Dot Hijau jika aktif/ada sisir, Abu-abu jika kosong).
вЂ‹2. Tab: Manage (Manajemen Mesin)
вЂ‹Tampilkan daftar mesin dari LIVE_TRACKING.
вЂ‹Fitur Tambah/Edit: Modal form untuk input No Mesin, SN, Jenis, dan Gedung.
вЂ‹Detail Mesin & Logika Pasang/Lepas:
вЂ‹Tombol "INSTALL REED" (jika mesin kosong): Update status di LIVE_TRACKING (isi ID sisir), update status di MASTER_STOK menjadi "Dipakai", lalu POST log ke HISTORY_PASANG.
вЂ‹Tombol "REMOVE REED" (jika ada sisir): Wajibkan user pilih Kondisi Akhir (BAIK/RUSAK). Update LIVE_TRACKING jadi kosong, update MASTER_STOK sesuai kondisi, lalu POST log ke HISTORY_LEPAS.
вЂ‹Riwayat: Gabungkan data dari HISTORY_PASANG dan HISTORY_LEPAS berdasarkan ID Mesin, urutkan descending, tampilkan 10 riwayat terakhir dalam tabel.
вЂ‹Tombol Cetak PDF (A4): Jalankan fungsi print (Instruksi detail di bawah).
вЂ‹3. Tab: Stok Sisir (Master Data)
вЂ‹Tampilkan list semua sisir dari MASTER_STOK dengan filter tab (Semua, Gudang, Dipakai, Rusak).
вЂ‹Tambah Sisir: Modal form data baru.
вЂ‹Detail Sisir (Kartu Stok): Klik item untuk melihat riwayat perjalanan sisir ini (pernah dipasang di mana saja) dengan menggabungkan log dari histori.
вЂ‹Service/Modifikasi: Tombol dengan 2 opsi:
вЂ‹Service Supplier: Update status di MASTER_STOK menjadi "Di Supplier".
вЂ‹Potong Sisir: Form input dimensi potong baru (Format: Destiny x Panjang x Tinggi). Update spesifikasi di MASTER_STOK dan catat riwayat pemotongan ke log.
вЂ‹Tombol Cetak PDF (A4): Jalankan fungsi print riwayat sisir.
вЂ‹4. Tab: History (Activity Log)
вЂ‹Ambil semua data dari HISTORY_PASANG dan HISTORY_LEPAS, gabungkan (merge), dan urutkan berdasarkan waktu/tanggal paling baru.
вЂ‹Tampilkan UI berupa timeline list memanjang ke bawah.
вЂ‹Gunakan badge warna: INSTALLED (Hijau), REMOVED (Merah), SERVICED/CUT (Biru/Kuning). Tampilkan detail: ID Mesin, ID Sisir, Operator, Tanggal, dan Kondisi.
